# Team docs — REGBOT notes

This folder is a **read-only mirror** of Mads's Obsidian notes for the REGBOT
balance assignment. It contains:

| File | What it is |
|---|---|
| `REGBOT Balance Assignment.md` | Progress log for the assignment — Tasks 1-4, design decisions, plots |
| `Lesson 10 - Unstable Systems and REGBOT Balance.md` | Lecture note covering unstable systems, Nyquist, post-integrator, gyro Lead |
| `PLAN.md` | Early-stage team plan (phases, role division) |
| `images/` | All plots generated by `simulink/regbot_mg.m` |

## Viewing

- **Plain Markdown** — any editor with Markdown preview (VS Code, etc.) works.
- **Obsidian** — open this `docs/` folder as a vault. Wikilinks like
  `[[Day 5 - Black Box Modeling]]` will show as broken because those notes
  live only in Mads's vault; they're informational pointers, not required.

## Staying in sync

Mads keeps `docs/` up to date with `scripts/sync_docs.sh` (run before
committing). Teammates should **not** edit Markdown files in `docs/`
directly — changes will be overwritten on the next sync. If you spot
something to change, message Mads or open a PR that only touches `src/`,
`simulink/`, `missions/`, or the team repo's top-level `README.md`.

### Running `regbot_mg.m` locally

When a teammate runs `simulink/regbot_mg.m`, fresh plots are written into
`docs/images/` so the Obsidian notes display **your own** results, not
Mads's last-committed ones. This produces uncommitted diffs — that's
expected.

**Don't** `git add docs/images/*.png` — the committed images are
Mads's canonical versions synced from his vault.
